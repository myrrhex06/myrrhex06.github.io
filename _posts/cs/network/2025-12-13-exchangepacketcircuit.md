호스트들 간에 메시지를 주고 받는 방식에는 아래와 같이 두 가지 방식이 있음.

- 회선 교환 방식
- 패킷 교환 방식

## **회선 교환 방식이란?**

회선 교환 방식은 호스트들 간에 메시지를 주고 받기 전 회선으로 먼저 연결이 되어 있어야함.

만약 연결되어 있지 않다면, 서로 메시지를 주고 받을 수 없음.

하나의 호스트가 다른 하나의 호스트와 회선을 연결하여 전송로를 점유했다면, 또 다른 호스트는 해당 전송로에 회선을 연결할 수 없음.

![image](/assets/img/circuitConstructure.png)

### **장점**

통신 시간 동안 전송로를 점유하기 때문에, 메시지 전달에 지연이나 밀림 현상이 발생하지 않아 시간당 전달되는 데이터 양이 일정함.

### **단점**

위에서 설명한 것 처럼 이미 점유된 전송로에는 다른 호스트가 연결할 수 없는데, 만약 해당 회선을 사용하는 호스트들끼리 메시지를 주고 받지 않는다면 이것은 자원 낭비가 됨.

## **패킷 교환 방식이란?**

패킷 교환 방식이란 메시지를 패킷이라는 단위로 나눠서 전송한 후 도착지에서 다시 재조립하여 사용됨.

> 패킷의 구성 요소로는 헤더, 페이로드, 트레일링이 있으며, 헤더에는 도착지의 주소, 페이로드에는 전달하려는 내용이 들어감.
{: .prompt-tip }

이 방식은 회선 교환 방식과는 달리 통신을 하는 호스트들이 전송로를 점유하지 않기 때문에, 네트워크 효율성이 좋음.

정해진 전송로로만 메시지를 송수신할 수 있는 회선 교환 방식과는 달리 패킷 교환 방식은 패킷 단위로 경로가 동적으로 결정됨.

호스트들 사이에 있는 패킷 스위치가 패킷들을 전송할 경로들을 설정함.

> 대표적인 패킷 스위치로는 스위치, 라우터가 있음.
{: .prompt-tip }
