[지난글](https://myrrhex06.github.io/posts/ethernetstandard/)에서 이더넷 표준에 따라 통신 매체의 전송 속도, 종류가 달라질 수 있다고 했음.

이더넷 표준에 맞춰 구현된 통신 매체를 표기할 때는 아래와 같은 형태로 표현함.

```
전송속도변조타입-추가특성
```

예시
```
2.5GBASE-T
```

### **전송 속도**
- 숫자만 적혀져 있을 경우 Mbps를 의미
  - ex. 100BASE-T
- G가 붙어 있을 경우 Gbps를 의미
  - ex. 2.5GBASE-T

### **변조 타입**
- 비트 신호로 변환된 데이터를 통신 매체로 전송하는 방식을 의미함.
- BASE(BASEband) 방식은 디지털 신호를 송수신하는 방식을 의미함.
  - 일반적인 LAN 환경에서는 대부분 BASEband 방식을 사용하고 있음.

> BASEband 외에도 BROADband, PASSband 방식이 있음.
{: .prompt-tip }

### **추가 특성**
- 통신 매체의 특성을 명시함.
- 여기에는 다양한 종류의 특성을 명시할 수 있음.
  - 케이블 종류
  - 전송 가능한 최대 거리
  - 물리 계층 인코딩 방식
  - 레인 수

> 물리 계층 인코딩 방식이란 데이터를 비트 신호로 변환하는 방식을 의미하며, 레인 수는 비트 신호를 옮길 수 있는 전송로의 수를 의미함.
{: .prompt-tip }

**케이블의 종류**

| 케이블                 | 표기법 |
| ---------------------- | ------ |
| 동축 케이블            | C      |
| 트위스티드 페어 케이블 | T      |
| 단파장 광섬유 케이블   | S      |
| 장파장 광섬유 케이블   | L      |

> 트위스티드 페어 케이블은 구리 선을 이용하며, 광섬유 케이블은 빛을 이용함.
{: .prompt-tip }

위에서 본 예시를 다시 보면서 설명해보자면.

예시
```
2.5GBASE-T
```

- **2.5G**: 2.5 Gbps 전송 속도를 지원 
- **BASE**: BASEband 변조 타입 사용
- **T**: 트위스티드 페어 케이블
